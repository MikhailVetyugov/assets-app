
Общее:
Напиши веб-приложение, которое будет вести учёт капитала человека относительно времени и рассчитывать его доходность.
Капитал представляет собой набор активов. Каждый актив имеет название, стоимость, вид, даты его изменения или создания, причину изменения.
Виды бывают следующими: акции, облигации, недвижимость, криптовалюта, золото, бизнес, авто, другое.
Это приложение облегчит жизнь, упростив подсчёты как минимум для моего капитала.
Пример: капитал состоит из двух активов. 
Первый актив: "авто Hyundai Sonata" вида "авто" стоимостью 20000$, создан 01/09/2023.
Второй актив: "Microsoft Inc." вида "акции" стоимостью 3000$, создан 02/11/2023 с ценой 2500$, изменен 12/12/2023 с ценой 3500$ по причине изменения цены, изменен 29/12/2023 с ценой 3000$ по причине изменения цены.
Функциональность:
- Первый запуск приложения. Создание активов.
Если данных о капитале нет (первый запуск приложения), то приложение просит создать активы. Когда пользователь создаёт актив, выводится модальное окно, предлагающее указать название актива, вид и стоимость.
Это модальное окно содержит кнопку "Сохранить", которая сохраняет эти данные. Приложение запоминает текущую дату, в которую пользователь создал активы для капитала.
В любую дату можно добавить новый актив, используя описанное выше модальное окно.
- Изменение стоимости:
После создания капитала в любую дату можно изменить один из активов капитала: удалить или изменить его стоимость. 
Если пользователь изменяет стоимость в отрицательную сторону, то нужно указать причину изменения: 1) изменилась цена актива; 2) выведены денежные средства из актива
Если пользователь изменяет стоимость в положительную сторону, то нужно указать причину изменения: 1) изменилась цена актива; 2) введены новые денежные средства в актив
При изменении капитала приложение запоминает дату (для будущих расчётов), в которую было произведено это изменение. Причины изменения также сохраняются.
Стоимость можно изменить в отрицательную сторону вплоть до 0.
- Удаление актива.
При удалении актива показываем модальное окно с кнопкой подтверждения и просим указать причину удаления:
1) актив обесценился, и его цена стала нулём 2)  выведены денежные средства из актива 3) актив добавлен по ошибке
После выбора причин удаления и нажатия на кнопку подтверждения: в случае причин 1 и 2 присваиваем под капотом нулевую цену, сохраняем причину удаления, а в случае причины 3 удаляем актив полностью безвозвратно. Таким образом, причины 1 и 2 - по сути как изменение актива, а не его удаление.
- Отображение капитала
Капитал отображаем в табличной форме. В первом столбце таблицы отображается список активов. Второй и последующий столбцы представляют собой цены на дату. Наверху столбца пишется дата, в которую было произведено изменение. Это даты, в которые проводилось изменение хотя бы одного актива.
Самая нижняя строка таблицы представляет собой сумму цен активов, т.е. это строка "Итого". Её выделяем жирным.
- Распределение активов
Под таблицей капитала отображаем информацию в формате ключ-значение, сколько какой вид активов занимает от капитала на сегодня
- Расчёт доходности
Под таблицей капитала показываем данные о доходности.
1) Показываем среднегодовую доходность всего капитала на последнюю дату, где были изменения
2) Показываем среднегодовую доходность для каждого вида активов на последнюю дату, где были изменения
Для второго случая используется формула средневзвешенной годовой доходности с учётом вводов/выводов. В гугл-таблицах такая формула называется XIRR, принимает денежные потоки и даты.
Если было проведено изменение в этом виде активов в меньшую сторону по причине вывода средств, рассматриваем дельту как отрицательный денежный поток,
если было проведено изменение в этом в виде активов в большую сторону по причине ввода средств, рассматриваем дельту как положительный денежный поток.
Пример работы с формулой:
11/15/2017, созданы активы вида "акции" с ценой 360000
08/15/2018, ввод денежных средств 374800 (активы вида "акции" стоят теперь 734800)
08/15/2018, ввод денежных средств 287500
08/15/2018, ввод денежных средств 200000
08/08/2018, ввод денежных средств 290000
03/15/2019, ввод денежных средств 559557
03/29/2019, ввод денежных средств 130000
07/09/2019 стоимость данного вида активов составляет 2470452 (ведь цена активов может измениться не только по причине ввода денежных средств в актив)
Тогда выводим доходность 14.45% (округляем до двух цифр после запятой).
- Валидация
Если пользователь не заполняет поля при создании или изменении или удалении актива, то выведи сообщение, что нужно заполнить. Если пользователь вводит отрицательную стоимость, то выведи сообщение,
что нужно вводить положительную стоимость.
- Технологии:
Для интерфейса используй React, Redux, Redux-Thunk, CSS modules, Typescript. Сборка с помощью webpack. Итог сборки содержит JS-бандл, CSS и HTML-файлы, которые будут потом загружены на CDN.
Можешь использовать дополнительные библиотеки на своё усмотрение.
Для сохранения данных используй localStorage. В будущем будет использоваться вместо localStorage база данных, поэтому инкапсулируй работу с localStorage в одном файле, чтобы легче было переехать на базу данных.
